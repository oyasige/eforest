
*eforest 概要

**目的
スケジュールパターンを管理する。
管理している情報を元にユーザにサービスを提供する。

**関連
[img:https://cacoo.com/diagrams/Y4PWL3XSx2hmBcwL-7AAC2.png]

1.ユーザはデバイス（PC,携帯）からeforestにアクセスできる。
2.ユーザはスケジュールパターンを登録、変更、削除できる。
3.eforestは2.を元に、スケジュールの登録、変更、削除機能をユーザに提供できる。
4.eforestは他WebService(google,yahooカレンダ等)と連携できる。
5.ユーザはユーザ間でスケジュールパターンを共有できる。

**例1
1−1.営業のAさんはある企業から受注をとった。
  　Aさんはiphoneからeforestにアクセスした。
1-2.eforestで受注を基準イベントとするスケジュールパターン
   「基準イベント、受注」「3日後、納品」「1週間後、請求」
    を登録した。
1-3.登録したスケジュールパターンを元に今回の受注、納品、請求の
    スケジュール(3つのイベント)を一括登録した。
1-4.Aさんはeforestのgoogle、yahooカレンダー連携設定を行っている。
    eforestは1-3の内容をAさんのgoogle、yahooカレンダーに登録した。
1-5.Aさんは営業のBさんにスケジュールパターンを登録したことを教えた。
    Bさんは他企業から受注を頂いた。
    BさんはeforestからAさんのスケジュールパターンを元に
    今回の受注、納品、請求のスケジュールを一括登録した。

**例2
2-1.高校で野球部に所属するCさんは1ヶ月後に野球大会に出場する予定である。
    CさんはAndroid携帯からeforestにアクセスした。
2-2.以前eforestで野球大会を起算日とするスケジュールパターン
   「起算日、野球大会」「2週間前〜、朝練」「1週間前、グラブお手入れ」「次の日、ユニフォーム洗濯」
　  を登録していた。
2-3.登録したスケジュールパターンを元に
　  今回の野球大会、朝練、クラブお手入れ、ユニフォーム洗濯のスケジュール(4つのイベント)を一括登録した。
2-4.Cさんはeforestのgoogleカレンダー連携設定を行なっている。
    eforestは2−3の内容をCさんのgoogleカレンダーに登録した。
2-5.CさんはチームメイトのDさんにスケジュールパターンを登録したことを教えた。
    DさんはeforestからCさんのスケジュールパターンを元に
　　新しいスケジュールパターン
　　「起算日、野球大会」「3週間前〜、朝練」「1週間前、グラブお手入れ」「次の日、ユニフォーム洗濯」
    を登録した。
